第五章 用户管理

5.1-用户与用户组管理

Linux系统支持多个用户在同一时间内登陆，不同用户可以执行不同任务，并且互不影响。

5.1.1-用户和用户组

用户组：建立一个组，让在这个组里面的人们有查看修改此文件的权限。

5.1.2-用户和用户组管理

UID:Linux只认识用户对应的ID，应户名与ID相对应的文档保存在/etc/passwd文件中。

GID：同上，相应数据保存于/etc/group文件中。

长格式显示了文件控制块内容，root和root表示文件拥有者和拥有群组 ，对应于用户UID和GID。真正在文件控制块中存放的是UID和GID。

5.2-用户和用户组管理相关文件

相关重要文件有passwd,shadow,group,gshadow

5.2.1-passwd

/etc/passwd属于系统用户配置文件，存贮系统中所有用户的基本信息，并且所有用户对其都有执行读操作。

0：root超级管理员用户

1~499：系统用户或伪用户（系统或服务正常运行所必须的用户）

500~：500以后的ID是留给一般用户的。

/etc/passwd文件每行的含义

用户名：密码：UID:GID：描述性信息：主目录：默认shell

初始组：用户登录时就有这个用户组的相关权限。

附加组：用户加入的其他用户组并拥有相应的权限。

主目录：拥有操作权限的家目录。

5.2.2-shadow文件

/etc/shadow :存储Linux中用户的密码信息，又称“影子文件”。

字段1：用户名

字段2：SHA512散列加密算法加密的密码，手动修改系统无法识别

字段3：最后一次修改时间，data -d “1970-01-01 15775 days”,系统命令转换为习惯日期。

字段4：最小修改时间间隔

字段5：密码有效期，默认99999天

字段6：密码需要变更前的警告天数，默认为7。

字段7：密码过期后宽限天数

字段8：账号失效时间

字段9：保留字段

若root账号的密码遗失，则需要重新启动系统，进入单用户模式，系统会提供root权限的bash接口，此时可以用passwd命令修改账户密码。

5.2.3-group文件

/etc/group:用户组配置文件。

组名：密码:GID:该用户组中的用户列表

想把某个用户加入该用户组，直接在最后面输入空格+用户名

例如 root：x：0：root lamp

5.2.4-gshadow文件

存储/etc/shadow里面的密码信息，只有root用户有权限

5.3-用户和用户组管理的命令

5.3.1-添加用户

[root@localhost~]#useradd [选项] 用户名

[root@localhost~]#useradd student

-u UID

-d 主目录

-c 用户说明

-g 初始组

-G 附加组

-s 登录shell

-e 用户失效日期，YYYY-MM-DD

-o 允许创建相同UID

-m强制建立用户主目录

-r 创建系统用户

在/etc/passwd ,/etc/shadow,/etc/group,/etc/gshadow,分别创建相应的数据，创建家目录/home/student和邮箱目录/var/spool/mail

同时还会将/etc/skel目录中的配置文件复制到新用户的主目录

useradd在添加用户时参考的默认值文件主要有两个，分别是/etc/default/useradd和/etc/login.defs

5.3.2-密码配置命令

[root@localhost~]#passwd [选项] 用户名

空：设置密码

-S 查看加密用户密码

-l 暂时锁定用户

-u 解锁用户

–stdin 通过管道符输出的数据作为用户的密码，用于批量添加用户

/root/.bash_history显示历史命令

注意两点，手动清除历史命令，和强制第一次登陆修改密码（chage）

5.3.3-修改用户信息

[root@localhost~]#usermod [选项] 用户名

-c ：修改用户说明信息

-d：修改用户主目录

-e：修改用户失效日期

-g：修改用户初始组

-u：修改用户UID

-G：修改用户附加组（原本的组却被移除，尽量用gpasswd）

-l：修改用户名称

-L：临时锁定用户，就是在密码字段前加！就不能登陆了

-U：解锁

-s：修改用户登陆shell

man usermod：查看支持命令

5.3.4-修改用户密码状态

[root@localhost~]#chage [选项] 用户名

-l：密码详细信息

[root@localhost~]#chage -d 0 student

批量添加用户时新用户第一次登录需要修改密码

5.3.5-删除用户

[root@localhost~]#userdel -r 用户名

-r：删除用户的同时删除家目录

因为系统可能产生文件，所以在删除用户时先find -user 用户名查找用户文件

5.3.6-查看用户UID GID

[root@localhost~]#id用户名

可以查看UID,GID,附加组

5.3.7-用户间切换

[root@localhost~]#su [选项] 用户名

-：不仅切换用户还切换环境变量，默认切换到root用户

su - root

-l：与-类似

-p：切换用户但不换环境

-m：同-p

-c：切换用户并执行一次命令后切换回来，后面通常接执行的命令

su - -c ‘useradd user1’ root

5.3.8-用户组管理

[root@localhost~]#groupadd [选项] 组名

-g GID

-r：创建系统群组

[root@localhost~]#groupmod [选项] 组名

-g GID

-n 新组名：修改组名

[root@localhost~]#groupdel 组名

不能用groupdel随意删除组，仅适用于不是任何用户初始组的组。

[root@localhost~]#gpasswd [选项] 组名

使用gpasswd给组设置一个组管理员

空：设置密码

-A user1，user2，…：将群组控制权移交

-M user1，user2，…：批量添加用户

-r：移除密码

-R：使密码失效

-a：添加用户进组

-d：移除用户

[root@localhost~]#newgrp组名

将用户附加组选择一个群组，作为用户新的初始组

第六章 文件管理

由于内存的断电易失性，计算机系统保存的程序，数据，等都是以文件形式存放在磁盘上的。所以对用户来说，所有信息，数据都只能以文件方式访问。

6.1-文件系统

系统角度：对文件存储设备的空间进行组织和分配，负责文件存储并对存入的文件进行保护和检索的系统。

6.1.1-文件系统的概念

Linux系统中文件是数据的集合，文件系统包含文件中的数据和文件系统的结构

文件：文件系统中存储数据的一个命名对象。用户眼中最小的逻辑存储单元。

目录：包含许多文件控制块项目的一类特殊文件。

子目录：包含一目录中的另一目录。

1.Linux不以文件扩展名区分文件类型。

IDE设备：/dev/hd[a-d]

SCSI/SATA/U盘：/dev/sd[]a-p

软驱：/dev/fd[0-1]

打印机：/dev/lp[0-15]

光驱：/dev/cdrom

鼠标：/dev/mouse

磁带机：/dev/st0或/dev/ht0

6.1.2-文件结构和类型

文件 = 数据项 + 文件控制块（索引节点）

Linux文件类型：

普通文件：

目录文件：内核组织文件系统的基本节点；

链接文件：指向一个真实文件的链接；

设备文件：Linux系统为外部设备提供一个标准接口，将外部设备视为一种特殊文件，通常存放于/dev目录下，根据访问数据方向不同，设备文件又可以分为块设备和字符设备

6.2-文件与目录操作

包括对文件和目录的浏览，创建，修改以及删除

6.2.1-文件操作命令

1.cat

显示文件内容

cat [选项] 文件名

合并文件

cat 文件1 文件2 > 文件3

-n：对输出的所有行进行编号

-b：只对非空行进行编号

\2. more（分屏显示）

more [选项] 文件名

less [选项] 文件名

\3. head

显示指定文件前若干行内容

-n k：k表示行数

\4. tail

与head相反

\5. touch

\6. grep

c*：匹配0个到多个字符c

. ：匹配任何一个字符

^:行首定位符

$:行尾定位符

\7. sed

流编辑器，完美配合正则表达式使用。主要用于自动编辑一个或多个文件，简化对文件的反复操作，编写转换程序。

\8. awk

数据处理工具，寻找有目标文本的行，如果找到就在该行执行用户想要的操作。

9.cp

10.rm

11.mv

12.sort

13.wc
